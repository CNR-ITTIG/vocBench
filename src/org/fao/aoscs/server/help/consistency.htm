Consistency

The Consistency Check Module allows editors and contributors to check the validity and the quality of the concept server data.

This module does not change data in the Workbench. The user checking the data decides if action should be taken on the data or not.

By clicking the arrow of the drop-down menu you can select the different types of consistency checks.

Some examples of consistency checks may include:
  • Identification of concepts which may be children of deprecated concepts.

  • Identification of concepts in the same hierarchy that do not also have associative relationships between them.

  • Verification of concepts which may have the same label in a particular language. This identifies problems such as reuse of a label as a non-preferred term and also as a preferred term, or same label as preferred term for two concepts (in this case the user may decide to rename one).

  • Verification on terms which do not have relationships: this may identify missing relationships

  • Verification of the status which may include:
    1. List of concepts already approved (published and validated) which have not been lexicalized in a particular language: this will identify possible missing translations.

    2. List concepts which do not have relationships: this may identify missing relationships

    3. List of unused relationships: this can help verify if a relationship is not applied or should just be removed as non pertinent to the selected domain.
