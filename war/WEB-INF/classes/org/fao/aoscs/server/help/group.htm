[ Non logged-in Users]
  • can navigate or search the system.
  • can submit suggested terms in any language.
  • can also do other kind of suggestions on concepts and relationships.
  • all suggestions made by these users should be associated with their IP address.
  • all suggestions will be identified as �guest proposed� and will be assesed by an authorized and registered validator.
  • guest proposals will be visible as such in the system. This does not hold for proposed deletions or edits of already existing terms and concepts. Such edits will only show up upon successful validation of a validator.
[Term editors (Terminologist/Thesaurus Editor)]
  • have full permissions on terms. which they own (i.e. created themselves).
  • this means that any changes on terms which they own (deleting or editing)
  • will be published without additional validation through a validator.
  • any other changes will be proposals that need to be validated.
  • if a term editor creates a new term in his language, this will be published without additional validation
  • cannot do concepts modifications.
  • concept modifications and modifications on terms which they are not the owner of will be set only as "proposed�.
  • every editor has set one or more languages and can add/edit/delete only terms of that Language (for other languages edit access should be prohibited).
  • can provide suggestions to more experienced users (e.g. concepts modifications that will  be processed by Ontology editors).
  • term editors who are the owner/creators of a term, can delete it completely;
[Ontology Editors (more experienced Terminologist/Thesaurus Editor)]
  • are Terminologist/Thesaurus Editor who know more about ontologies and the distincion of concepts and terms .
  • can manage terms (of specific languages, i.e. the ontolgoy editors registered languages) and concepts.
  • if an ontology editor creates a new concept, it will be published without further validation.
  • only edits on concepts of which the ontology editor is not the owner will enter the system as proposal and will have to be validated. 
  • if an ontology editor deletes a concept, only the terms of the corresponding languages will be set to �proposed deprecated� and the concept status will be set to �proposed deprecated�.
  • can completely delete the concept, if the ontology editor is the owner/creator of the concept and and only if he has privileges in all the languages of the terms of the concept.
[Validator]
  • are experts that can check and validate in their respective languages the work done by the editors (terms and ontology editors) and guests.
  • they validate all proposals and guest proposals and have the power to reject proposals, i.e. remove them again from the model or reset them to their previous state (in case of edits)
[Publisher (Ontology Editors)]
  • have permission on terms and concepts.
  • can validate editors and confirm validator work.
  • they ultimately approve a validated entry and change its status to "published"
  • they can still reject validated entries and thus remove them again from the model (in case of new proposals) or reset them to their previous status (in case of an edit or delete proposal).
  • access to all languages.
[Administrators]
  • have all the above permission.
  • have the permission to change manually the status of a concept or a term (for all other users a status will be set by the system automatically based on the action)
  • can manage users and groups.
  • can provide statistics.
  • access to all languages.


# Add Group

Step 1: Click [New] button to add new group.

  • Opens a new group management window.
Add new group

Group name :            <name of the group>
Group description :    <description of the group>
 
   [Add] [Close]

Step 2: Enter the new group name.

Step 3: Enter the description for the new group to be created.

Step 4: Click [Add] button to add new group or [Close] button to cancel the operation.
  • <Insert new group complete> message will be displayed if the add group operation is successful.
  • <Please enter group name> message will be displayed if the group name is empty.


# Delete Group

Step 1: Select the group to be deleted from the list of existing group.

  • Confirmation window will popup.
Do you really want to delete this group?

[Ok] [Cancel]

 Step 2: Click [Ok] button to confirm group deletion or [Cancel] button to cancel the delete operation.

  • <Remove groups complete> message will be displayed if the group deletion operation is successful.
  • <Group data use in db can not delete> message will be displayed if the users are assigned to that group and the group is not empty.
  • <Can not delete main group!> message will be displayed if default group is selected.

# NOTE:  [Administrator], [Guest], [Ontoeditor], [Publisher], [Termeditor], [Validator] are default group which     cannot be deleted.


# Assign User

Step 2: Click [Add] button to assign user to the selected group.

# Remove User

Step 1: Select the user from the list of users assigned to the group.
Step 2: Click [Remove] button to remove the selected user from the selected group.

- Confirmation window will popup.

Do you want to remove user from the group?

[Ok] [Cancel]

Step 3:  Click [Ok] button to confirm user removal from the group or [Cancel] button to cancel the remove operation.
  • <Remove user from the group complete> message will be displayed, if the user is removal from the group is successful.
  • <No select user!> message will be displayed if the user is not selected.
